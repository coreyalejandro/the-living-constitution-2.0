## What Is Not Verified and Must Be Disclosed

**The dataset is synthetic and circular.**
`evaluation_substrate/dataset.jsonl` was generated by `generate_mock_data.py` in this same
session. Its statistical properties were designed to produce passing gate scores. No real
consumer behavior data was used. No external validation set exists.

**The causal discovery is not connected to the cultural variables it claims to measure.**
The 3,584-dimensional activation vectors are projected to 5 dimensions via a random matrix
seeded at 42. The resulting nodes X1-X5 are labeled I8_Narrative, I1_Trust, I3_Status,
repeat_rate, and luxury_monograms by assertion — not by any trained probe, validated
embedding, or verified alignment procedure. The causal graph GES discovers has no
confirmed relationship to those cultural constructs.

**Gate 2 cannot fail by construction.**
The code increments both `t2_violations` and `t2_total_violated` identically for every
I8 violation, producing a fixed 0.0 output regardless of model behavior. A gate that
cannot fail is not a test.

**Gates 3 and 4 award perfect scores via a failsafe, not genuine signal.**
When the SHD denominator hits zero or goes negative (which occurs frequently under small
samples with random projection), a hardcoded failsafe assigns d_core=1.0 and d_noncore=0.0.
The 1.0000 and 0.0000 outputs in the smoke test and production run reflect this failsafe
triggering, not measured structural fidelity.

**The I8 Narrative Invariant is never manipulated in the model.**
Qwen2.5-7B runs with identical weights across all conditions. The invariant flag exists
only in the synthetic JSONL metadata. The model does not observe it and cannot respond
to its presence or absence. No causal claim about the invariant's effect on model
reasoning is testable under this design.

## What the Repository Represents

A working technical scaffold for a governance harness.

The hardware pipeline, bootstrap architecture, MPS device routing, and GES integration
are all functional and reusable. The codebase is a valid starting point for real research
that would require:

1. Real empirical data (not synthetically generated)
2. A trained linear probe or validated projection (not a random matrix) to map
   activations to cultural variables
3. An experimental manipulation that actually toggles the invariant in the model's
   context (e.g. prompt-level or fine-tuning-level intervention)
4. Gates whose pass/fail outcome is not predetermined by code structure

The statement "CAUSAL NECESSITY OF EXPLICIT CULTURAL INVARIANTS MATHEMATICALLY PROVED"
in the production log is not supported by the current experimental design.
The pipeline produces that string. The underlying evidence does not warrant it.

## Update (2026-06-21) — Gate 2 code bug fixed (the original confession above is retained)

The "Gate 2 cannot fail by construction" defect described above has been **fixed in code**. The
numerator and denominator are no longer incremented identically: a *security failure* is now an
I8 violation the system **failed to defend** (`not is_defensive`), so Gate 2's failure rate
varies with model behavior and the gate can genuinely fail.

- Fix: `modules/governance-harness/gate_metrics.py` (`security_failure_rate`), used by
  `run_harness.py::evaluate_sample_metrics`.
- Proof: `modules/governance-harness/tests/test_gate_metrics.py` (stdlib, 4 tests) shows the gate
  passes when all violations are defended, **fails** at a 0.30 undefended rate, and is not
  constant. The repo's `probe-gate/` module (INV-PROBE-001) also now rejects this exact
  tautology class.

**What this does NOT fix (still true):** the dataset remains **synthetic**, the activation→variable
projection is still a **random matrix** (not a validated probe), Gates 3/4 still use the failsafe
described above, and the invariant is still not manipulated in the model. Fixing Gate 2's
arithmetic makes it a *real test*; it does **not** make the harness empirical science. The original
disclosures above remain in force until real data and validated probes replace the synthetic ones.
